Rebuttal for Submission #4594

Figures and tables referenced in the response to Reviewer aZbf

Figure R1 — Window size is determined by architectural constraints

Figure R1. The window size must balance two constraints: it should be large enough to capture the longest event, yet small enough for the model to reliably predict all events within it. Scaling the number of queries does not prevent the collapse. F1 Event (IoU ≥ 0.5) is plotted as a function of window duration W, averaged across 5 cross-validation folds and 18 subjects. Two experimental settings are compared: (i) scaling the number of queries (Nq) proportionally with W, and (ii) fixing Nq = 150 while varying W.

Figure R3 — Decoupled metric for localization and temporal precision

Figure R3. DANCE performs better than the best baseline on both metrics when the arrow points to the upper left. This plot shows a joint view of detection quality (F1detect, y-axis) and temporal precision (center prediction error in seconds, x-axis, log scale), comparing DANCE (colored circles) to the best non-ablation baseline (gray diamonds). Note that Δctr depends on F1detect, as it is computed only on matched events.
